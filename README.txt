PHP Testing System позволит тренерам осуществлять тестирование слушателей по различными дисциплинам.

Права:
Администратор - Имеет право выполнять любые операции без каких бы то ни было ограничений.
RD-менеджер - Отвечает за создание в PTS направлений обучения и тестов.
Тренер - Отвечает за создание в PTS групп слушателей, назначение группам слушателей тестов.
Слушатель - Проходит тестирование по назначенным тестам.
Гость - Особый вид слушателя, для прохождения теста не авторизующийся в системе, а использующий специальный одноразовый ключ.


«Тест» – набор вопросов, специально подготовленный для назначения к прохождению. Отображается в списке тестов.
Вопросы имеют свою категорию(PHP, Java и т.д.) и подкатегорию(PHP - основы, PHP - ООП и т.д.)
Каждый тест имеет только одну категорию и подкатегорию.
Каждый вопрос может находиться в нескольких категориях и подкатегориях.

Администраторы создают группы и добавляют туда слушателей, потом этой группе назначается тест. И каждый слушатель из группы должен пройти этот тест.


На данном этапе реализованно:
- Авторизация.
- Админка в которую входит: создание групп, вопросов(2 вида), добавление прав пользователям, создание категорий и подкатегорий, сопоставление вопросов и категорий/подкатегорий, создание тестов, просмотр сущ-х тестов,
просмотр статистики(общая, по группам и все пройденные тесты пользователя), просмотр тестов, просмотр вопросов по категориям, добавление/удаление пользователей из групп и др.
- Личный кабинет(различен для администратора и пользователя).
- Изменение различных параметров пользователя(ФИО, email).
- Прохождение тестов для слушателей(есть возможность продолжить незаконченный тест).
- Просмотр статистики по своим пройденным тестам(для пользователей).
